# EmailParser - сортировка входящих писем

Автоматически читает письма из папки `inbox`, определяет их категорию по ключевым словам, распределяет все письма по категориям по отдельным папкам с названиями категорий в папку `output`. Также формируется лог обработки писем

## Структура проекта
- `Email.py` - датакласс, который представляет одно письмо
- `EmailParsing.py` - парсинг файлов писем по расширениям (.txt, .json, неизвестные и поврежденные письма)
- `Classificator.py` - классификация письма по катеогрям опираясь на ключевые слова
- `Sorter.py` - сортировщик писем по папкам категорий
- `Statistics.py` - запись статистики по обработанным письмам(все письма, поврежденные и по категориям)
- `main67.py` - главный модуль, запускает весь пайплайн
- `run.sh` - Bash-скрипт запуска

## Как запустить
Для запуска нужно положить входящие письма в папку `inbox` и запустить Bash-скрипт : 

``` bash
./run.sh
```
### Для запуска на Windows (или если включен Line Separator CRLF(/r/n))

```bash
sed -i '' 's/\r//' run.sh
```
Далее выполнить обычную программу запуска
**После выполнения программы папка `inbox` станет пуста. Все письма будут раскинуты по папкам своих категорий в папке `output`. Лог будет видет в `logfile.log`**

## Категории писем

| Категория | Описание |
|---|---|
| spam | Спам и фишинг |
| critical_incidents | Критические инциденты и сбои |
| monitoring | Автоматические уведомления мониторинга |
| access_control | Запросы доступа и новые сотрудники |
| hardware | Проблемы с оборудованием |
| software_failures | Сбои программного обеспечения |
| documents | Документооборот и финансы |
| HR | Кадровые вопросы и встречи |
| unknown | Не удалось определить категорию |
| corrupted | Нечитаемые файлы |
